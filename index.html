Chip-8 Script

04/11/2018

Today I finished the first executable version of my little c8s-compiler (c8s = Chip-8 script). It converts a tiny custom scripting language called Chip-8 Script into chip-8 machinecode. C8s has a very limited set of features, since on the one hand Chip-8 has only 35 instructions of its own and on the other hand, my goal was to generate as little overhead as possible through low abstraction.

Currently supported features:

Create variables with the keyword var. There can only be a maximum of 15 total due to Chip-8's limitations. 

VAR a = 10
VAR b = a

If-statements can have the following forms of condition: V[x]==n, v[x]!=n, v[x]==V[y] 

IF a == 0:
  ...
ENDIF

For-loops define an index-variable that counts to the value behind to and increases by the value behind step. 

FOR i=0 TO 10 STEP 2:
  ...
ENDFOR

TinyTemplatesJS

TinyTemplates is a tiny templating engine in javascript that lets you create reusable, reactive components that can be rendered and updated in the DOM. The templates are using standard HTML syntax combined with a few additional nodes that work as statements (if) or loops (for, foreach). Data, that needs to be reactive, is stored as state that embeds in the view to get updated if any changes are made to the data. diffDOM is used to detect and patch these changes. Please have a look at the example below for a better understanding of the syntax.

Example

let counter = new TinyTemplate( 
    // Name
  "counter", 
  { // State
    number: 0 
  },
  { // Methods
    increaseNumber: function() {
      this.changeState({ number: this.getState("number") + 1 });
    },
    reset: function() {
      this.changeState({ number: 0 });
    }
  },// Template-view
  `<div class="counter>
      <fieldset>
        <legend><b>Counter</b></legend>
        <p>Counting: {{number}}</p>
        <button on-event="onclick" call="increaseNumber">Increment</button>
        <button on-event="onclick" call="reset">Reset</button>
        </fieldset>
    </div>`
);

// Mount the template to a root node to add it to the DOM.
counter.mount(document.getElementById("root-node-id"));

Overview

Mustaches

The identifiers inside double mustaches are replaced by their corresponding value in the state-object. If the parser can not find a corresponding value in the state-object, it searches for local variables with the same name and replaces it. It is also possible to access the properties of those variables: 
<p>Access: {{stateObject}}</p>
<p>Access: {{localVariable}}</p>
<p>Access: {{localVariable.property}}</p>

State-changes

If you want to change the value of a variable from the state-object (e.g. `name`), you need to call this.changeState({name: 'NewName'}). This way the engine registers the specific updates to the data to trigger a reparsing and change the parts of the DOM that are affected by the updates.

this.changeState({ number: this.getState("number") + 1 });

If

The body of an if statement only gets rendered when the condition (expr) inside the braces (this.getState('payment') === 'visa') evaluates to true. The statement ends with a /fi. 

<if expr="{{payment}} === 'visa'" )>
  <p>VISA was selected!</p>
</fi>

For

Use for loops to render nodes multiple times. The syntax should be self explaining. The first attribute var declares the index variable, from and to define the range of the loop and step is used to determine the number by which the index variable increments each cycle. The index variable (j in our case) can be accessed from anywhere inside the loop using 'mustaches'. 

<for var="j" from="0" to="{{max}}" step="2">
  <p>Counting: {{ j }}</p>
</for>

Foreach

The foreach loop is the range based counterpart to the standard for loop. It takes up to four attributes of which at least two must be provided. The elem attribute, which contains the current list item (or contains the key to the variable that contains the current list item to be more precise) and the list itself, specified in in. The other two optional attributes are idx that stores the index of the current item and arr, which contains the array itself. 

<foreach elem="elem" idx="i" arr="arr" in="this.getState('animals')">
  <div class="cart-product">{{fruit.product}}</div>
  <div class="cart-price">{{fruit.price}}</div>
  <br>
</foreach>
